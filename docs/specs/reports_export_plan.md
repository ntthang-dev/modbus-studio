# Implementation Plan: Reports Module (PDF/CSV Exports)

## Overview
Implement a dual-mode reports exporter in Modbus Studio. This enables SCADA operators to manually customize and export PDF/CSV shift reports, and permits Javascript automation scripts to execute headless scheduled report generations directly to the local file system.

---

## Architecture Decisions
- **PDF Layout Generator**: Standardize on `pdf: ^3.12.0` for report layouts, featuring corporate SCADA headers, tabular logs, and color-coded status elements.
- **Native File Dialog**: Use the `printing` package (`printing: ^5.14.3`) to open the macOS native share/save sheet, delegating OS-level writing to sandboxed system calls.
- **Headless Save Directory**: Automatic exports generated by QuickJS scripts will default to saving files under `~/Documents/ModbusStudio/Reports/` using standard Dart `File.writeAsBytes` channels.
- **SQLite Range Querying**: We will expand the Rust SQLite reader to retrieve logs within specific timestamp bounds to optimize query speed and memory footprint.

## Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| **File Sandbox Lockout** | High | Automated exports writing directly to `~/Documents/` might fail if macOS Sandbox is not configured with `com.apple.security.files.user-selected.read-write`. We fallback to prompting the user via `printing` if native write permissions are blocked. |
| **Large SQLite Rows memory spike** | Medium | Querying massive logs can freeze the VM. We apply an upper limit (e.g. 5000 rows max) and query in small chunks using offset limits if needed. |
| **UI Freeze during layout** | Medium | PDF layout calculation is expensive. We compute it asynchronously using background Futures or Dart isolates. |
